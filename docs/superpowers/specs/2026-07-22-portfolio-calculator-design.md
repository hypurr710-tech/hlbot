# Portfolio 현금흐름 계산기 (`/calculator` 전면 개편) 설계

날짜: 2026-07-22
상태: 승인됨

## 목적

단발 "금액×APR" 계산기를 **포트폴리오 현금흐름 대시보드**로 교체한다.
자산 항목을 등록해두면 포트폴리오 전체의 가중 이자율과 월/연 현금흐름이 집계된다.

## 항목 3유형

| 유형 | 입력 | 계산 |
|---|---|---|
| 이자형 (예금·스테이블·채권) | 이름, 통화(KRW/USD), 원금, 연이율% | 연 = 원금×이율, 월 = 연/12 |
| 고정수입형 (근로소득·월세) | 이름, 통화, 월 금액, 원금(선택) | 월 그대로, 연 = ×12 |
| 펀딩파밍 (라이브) | 버튼으로 추가, 이율 수동 오버라이드 가능 | 원금 = HL 예치금+현물 원금+기타 조정(실시간, /arb/history와 동일 정의), 이율 = 현재 펀딩률 기반 예상 APR |

## 계산 규칙

- 표시 기준 통화 KRW. USD 항목은 하나은행 환율로 환산 (총 자산엔 $ 병기).
- 가중평균 APR = (원금 있는 항목들의 연 현금흐름 합) ÷ (원금 합) × 100.
  원금 없는 고정수입(근로소득)은 현금흐름 합계에는 포함, APR 분자·분모에서 제외 (왜곡 방지).
- 모든 값 세전(gross).
- 펀딩파밍 예상 APR = Σ(숏 노셔널×시간당 펀딩률)×8760 ÷ 투입자본 ×100 (전체자본 기준).

## 화면 (위→아래)

1. 요약 카드 4: 총 자산(₩, $ 병기) · 가중평균 APR · 월 현금흐름 · 연 현금흐름
2. 항목 테이블: 이름 | 유형 | 원금 | 연이율 | 월 | 연 | 삭제. 펀딩파밍 행 "자동" 배지 + 실시간
3. 추가 폼: 유형 선택식 필드 전환. 이자형에 기존 "종목 APR 불러오기"(스캐너 실시간 APR) 유지
4. Sidebar 메뉴명 Calculator → Portfolio

## 에러/엣지

- 환율 미조회: USD 항목 "환율 대기", 합계에서 제외하지 않고 KRW 항목만으로 우선 표시
- 펀딩 데이터 미조회: 펀딩파밍 행 "—", 합계에서 제외
- corrupt localStorage → [] fallback (기존 store 패턴)
